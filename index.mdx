Grim is an open-source Minecraft anticheat built around a **1:1 movement simulation engine** — it predicts every possible player movement server-side and flags anything that deviates. Fully async and multithreaded, it scales to hundreds of concurrent players without impacting server performance.

## Why Grim?

<CardGroup cols={2}>
  <Card title="Prediction engine" icon="brain">
    Grim simulates every possible client movement server-side. Cheats cannot bypass a system that already knows what you *should* do.
  </Card>
  <Card title="Async & multithreaded" icon="bolt">
    All checks and the vast majority of listeners run on the netty thread — no main-thread impact, scales to hundreds of players.
  </Card>
  <Card title="Lag compensation" icon="clock">
    World changes are queued until they reach the player, so breaking blocks or teleporting never produces false positives.
  </Card>
  <Card title="1.8–1.21 support" icon="cubes">
    Every Minecraft version, every platform — Bukkit, Paper, Folia, and Fabric — with full cross-version physics accuracy.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Download Grim">
    Grab the latest build from [Modrinth](https://modrinth.com/plugin/grimac) (recommended) or use the nightly GitHub artifacts for Bukkit or Fabric.
  </Step>
  <Step title="Install the plugin">
    Drop the `.jar` into your server's `plugins/` (Bukkit/Paper/Folia) or `mods/` (Fabric) folder and restart.
  </Step>
  <Step title="Review configuration">
    Open `plugins/GrimAC/config/en.yml` and `punishments/en.yml` to tune thresholds and punishment actions for your server type.
  </Step>
  <Step title="Enable alerts">
    Run `/grimac alerts` in-game to subscribe to real-time cheat alerts. Optionally configure Discord webhook integration for off-server notifications.
  </Step>
</Steps>

<Note>
  Grim requires **Java 17 or higher** at runtime. Java 21 is required to compile from source.
</Note>
